## Responsible Disclosure Guidelines
- Provide a clear description of the issue  
- Include steps to reproduce if possible  
- Allow maintainers time to investigate and fix the issue  
- Do not exploit vulnerabilities or access unauthorized systems  

---

## Secret & Token Management
- Never commit secrets, API keys, or credentials to the repository  
- Always use GitHub Secrets for CI workflows  
- Sensitive repos must follow the principle of least privilege  
- Regular audits are recommended to ensure no secrets are leaked  

---

## Security Expectations
Contributors and AI agents must:
- Follow safe development practices  
- Avoid introducing insecure patterns  
- Validate changes through Issues/PRs  
- Use attestation logs for critical changes  
- Report any suspicious behavior immediately
