## 4. Release Workflow (Full Sequence)

1. Contributor opens PR


2. Athena checks structure


3. CI runs tests + lint + build


4. Raasid validates evidence


5. Aya reviews


6. Ashraf (if critical) reviews


7. Merge into main


8. Aya initiates Release


9. Evidence generated by CI


10. CAP event created


11. Raasid generates ledger entry


12. Release becomes immutable
