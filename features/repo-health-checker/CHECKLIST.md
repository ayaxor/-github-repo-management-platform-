
# Repository Health Checklist

This checklist defines the **baseline criteria**
used by the Repo Health Checker to evaluate repositories.

Checks are informational and non-enforcing.

---

## Documentation Checks

- [ ] README.md exists
- [ ] README.md describes project purpose
- [ ] README.md includes usage or setup section

---

## Governance Files

- [ ] LICENSE file exists
- [ ] CONTRIBUTING.md exists
- [ ] CODE_OF_CONDUCT.md exists (optional but recommended)
- [ ] SECURITY.md exists (recommended)

---

## Repository Structure

- [ ] Clear root-level structure
- [ ] No excessive files in root directory
- [ ] Logical folder naming conventions

---

## Project Metadata

- [ ] Repository description is set
- [ ] Repository topics/tags are defined
- [ ] Default branch is clearly defined

---

## Maintenance Signals

- [ ] Recent commits exist
- [ ] Issues or discussions enabled
- [ ] Clear ownership or maintainer info

---

## Notes

- Failing a check does NOT indicate poor quality
- Results are meant to guide improvement
- Checklist can evolve over time
